Transaksi mewakili pembayaran yang telah selesai atau sedang berlangsung melalui tautan pembayaran dan sesi checkout Anda. API transaksi bersifat read-only -- transaksi dibuat secara otomatis ketika pelanggan membayar melalui alur checkout Anda.

<Info>
  Jumlah transaksi disimpan dalam unit terkecil token (mis., 6 desimal untuk USDC). Bidang `amount_usd` menyediakan setara USD pada saat transaksi.
</Info>

## Objek Transaksi

<ResponseField name="id" type="string">
  Pengenal unik untuk transaksi (mis., `tx_abc123`).
</ResponseField>

<ResponseField name="status" type="string">
  Status saat ini dari transaksi. Salah satu dari: `pending`, `confirming`, `completed`, `failed`.
</ResponseField>

<ResponseField name="amount" type="string">
  Jumlah transaksi dalam unit terkecil token.
</ResponseField>

<ResponseField name="amount_usd" type="string">
  Setara USD dari jumlah transaksi pada saat pembayaran.
</ResponseField>

<ResponseField name="token_address" type="string">
  Alamat kontrak dari token yang diterima.
</ResponseField>

<ResponseField name="chain_id" type="number">
  Chain ID dimana pembayaran diterima.
</ResponseField>

<ResponseField name="tx_hash" type="string | null">
  Hash transaksi on-chain. Null selama `pending`.
</ResponseField>

<ResponseField name="payer_address" type="string">
  Alamat dompet dari pembayar.
</ResponseField>

<ResponseField name="recipient_address" type="string">
  Alamat dompet yang menerima pembayaran.
</ResponseField>

<ResponseField name="customer_id" type="string | null">
  ID pelanggan yang terkait, jika pembayar adalah pelanggan yang dikenal.
</ResponseField>

<ResponseField name="payment_link_id" type="string | null">
  Tautan pembayaran yang menghasilkan transaksi ini.
</ResponseField>

<ResponseField name="session_id" type="string | null">
  Sesi checkout yang menghasilkan transaksi ini.
</ResponseField>

<ResponseField name="source_chain_id" type="number | null">
  Jika pembayar membayar dari chain yang berbeda, ini adalah chain tempat mereka membayar.
</ResponseField>

<ResponseField name="source_token_address" type="string | null">
  Jika pembayar membayar dengan token yang berbeda, ini adalah token yang mereka bayar.
</ResponseField>

<ResponseField name="source_amount" type="string | null">
  Jumlah yang dikirim pembayar (sebelum swap/bridge), dalam unit terkecil token sumber.
</ResponseField>

<ResponseField name="fee_amount" type="string | null">
  Jumlah biaya platform dalam unit terkecil token yang diterima.
</ResponseField>

<ResponseField name="form_data" type="object | null">
  Data formulir kustom yang dikirimkan oleh pelanggan selama checkout.
</ResponseField>

<ResponseField name="shipping_address" type="object | null">
  Alamat pengiriman yang disediakan oleh pelanggan.
  <Expandable title="field alamat pengiriman">
    <ResponseField name="name" type="string">Nama penerima.</ResponseField>
    <ResponseField name="line1" type="string">Baris alamat 1.</ResponseField>
    <ResponseField name="line2" type="string | null">Baris alamat 2.</ResponseField>
    <ResponseField name="city" type="string">Kota.</ResponseField>
    <ResponseField name="state" type="string">Negara bagian atau provinsi.</ResponseField>
    <ResponseField name="postal_code" type="string">Kode pos atau ZIP.</ResponseField>
    <ResponseField name="country" type="string">Kode negara dua huruf.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="shipping_option" type="object | null">
  Opsi pengiriman yang dipilih.
  <Expandable title="field opsi pengiriman">
    <ResponseField name="id" type="string">ID opsi pengiriman.</ResponseField>
    <ResponseField name="name" type="string">Nama opsi pengiriman.</ResponseField>
    <ResponseField name="amount" type="string">Biaya pengiriman.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="metadata" type="object | null">
  Metadata dari sesi checkout.
</ResponseField>

<ResponseField name="created_at" type="string">
  Cap waktu ISO 8601 ketika transaksi dimulai.
</ResponseField>

<ResponseField name="confirmed_at" type="string | null">
  Cap waktu ISO 8601 ketika transaksi mencapai status `confirming`.
</ResponseField>

<ResponseField name="completed_at" type="string | null">
  Cap waktu ISO 8601 ketika transaksi mencapai status `completed`.
</ResponseField>

### Status Transaksi

| Status | Deskripsi |
|--------|-------------|
| `pending` | Transaksi dimulai, menunggu pengajuan on-chain. |
| `confirming` | Transaksi diajukan on-chain, menunggu konfirmasi blok. |
| `completed` | Transaksi dikonfirmasi dan pembayaran diterima oleh penerima. |
| `failed` | Transaksi gagal karena revert, timeout, atau dana tidak cukup. |

## Daftar transaksi

Ambil daftar transaksi yang dipaginasi dengan filter opsional.

<ParamField query="status" type="string">
  Filter berdasarkan status: `pending`, `confirming`, `completed`, `failed`.
</ParamField>

<ParamField query="customer_id" type="string">
  Filter berdasarkan ID pelanggan.
</ParamField>

<ParamField query="payment_link_id" type="string">
  Filter berdasarkan ID tautan pembayaran.
</ParamField>

<ParamField query="from" type="string">
  Filter transaksi yang dibuat pada atau setelah tanggal ISO 8601 ini (mis., `2026-01-01`).
</ParamField>

<ParamField query="to" type="string">
  Filter transaksi yang dibuat pada atau sebelum tanggal ISO 8601 ini (mis., `2026-02-28`).
</ParamField>

<ParamField query="page" type="number" default={1}>
  Nomor halaman untuk paginasi.
</ParamField>

<ParamField query="limit" type="number" default={20}>
  Jumlah hasil per halaman (maks 100).
</ParamField>

<CodeGroup>
```bash cURL
curl -X GET "https://platform-api.anyspend.com/api/v1/transactions?status=completed&from=2026-02-01&to=2026-02-28&page=1&limit=20" \
  -H "Authorization: Bearer asp_xxx"
```

```typescript SDK
import { AnySpend } from "@b3dotfun/sdk";

const anyspend = new AnySpend(process.env.ANYSPEND_API_KEY!);

const { data, pagination } = await anyspend.transactions.list({
  status: "completed",
  from: "2026-02-01",
  to: "2026-02-28",
  page: 1,
  limit: 20,
});
```
</CodeGroup>

## Ekspor transaksi

Ekspor data transaksi dalam format CSV atau JSON. Respons dikirimkan sebagai unduhan file.

<ParamField query="format" type="string" required>
  Format ekspor. Salah satu dari: `csv`, `json`.
</ParamField>

<ParamField query="status" type="string">
  Filter berdasarkan status: `pending`, `confirming`, `completed`, `failed`.
</ParamField>

<ParamField query="from" type="string">
  Filter transaksi yang dibuat pada atau setelah tanggal ISO 8601 ini.
</ParamField>

<ParamField query="to" type="string">
  Filter transaksi yang dibuat pada atau sebelum tanggal ISO 8601 ini.
</ParamField>

<Info>
  Ekspor dibatasi hingga 50.000 catatan. Untuk dataset yang lebih besar, gunakan filter rentang tanggal untuk membagi ekspor menjadi bagian yang lebih kecil.
</Info>

<CodeGroup>
```bash cURL
# Ekspor transaksi yang selesai untuk Februari 2026 sebagai CSV
curl -X GET "https://platform-api.anyspend.com/api/v1/transactions/export?format=csv&status=completed&from=2026-02-01&to=2026-02-28" \
  -H "Authorization: Bearer asp_xxx" \
  -o transactions.csv

# Ekspor sebagai JSON
curl -X GET "https://platform-api.anyspend.com/api/v1/transactions/export?format=json&status=completed" \
  -H "Authorization: Bearer asp_xxx" \
  -o transactions.json
```

```typescript SDK
// Ekspor sebagai aliran CSV
const csvStream = await anyspend.transactions.export({
  format: "csv",
  status: "completed",
  from: "2026-02-01",
  to: "2026-02-28",
});

// Ekspor sebagai array JSON
const transactions = await anyspend.transactions.export({
  format: "json",
  status: "completed",
});
```
</CodeGroup>
