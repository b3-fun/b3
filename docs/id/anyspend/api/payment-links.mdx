Tautan pembayaran adalah URL yang dapat dibagikan yang memungkinkan pelanggan Anda membayar dengan token apa pun di rantai yang didukung. Setiap tautan dapat disesuaikan dengan branding, formulir, opsi pengiriman, dan lainnya.

<Info>
  Tautan pembayaran secara otomatis menangani perutean token, abstraksi gas, dan pertukaran lintas-rantai melalui protokol AnySpend.
</Info>

## Parameter URL

Saat membagikan URL tautan pembayaran, Anda dapat menambahkan parameter kueri untuk mengisi data sesi sebelumnya, melacak atribusi, dan mengaitkan pembayaran dengan pelanggan Anda.

| Parameter | Deskripsi | Contoh |
|-----------|-------------|---------|
| `client_reference_id` | ID referensi internal Anda (ID pesanan, ID pengguna, dll.) | `?client_reference_id=order_123` |
| `metadata[key]` | Metadata kunci-nilai sembarang (notasi kurung, hingga 50 kunci) | `?metadata[user_id]=clerk_abc&metadata[plan]=pro` |
| `session_id` | Gunakan sesi checkout yang telah dibuat sebelumnya alih-alih membuat yang baru | `?session_id=cs_abc123` |
| `success_url` | Timpa URL pengalihan setelah pembayaran berhasil | `?success_url=https://example.com/thanks` |
| `cancel_url` | Timpa URL pembatalan/kembali | `?cancel_url=https://example.com/cart` |
| `utm_source` | Sumber UTM untuk atribusi | `?utm_source=twitter` |
| `utm_medium` | Media UTM untuk atribusi | `?utm_medium=social` |
| `utm_campaign` | Kampanye UTM untuk atribusi | `?utm_campaign=launch` |

### Batasan Metadata

- Maksimum 50 kunci per sesi
- Panjang kunci: hingga 40 karakter
- Panjang nilai: hingga 500 karakter

### Contoh: mengaitkan pembayaran dengan pelanggan Anda

```
https://anyspend.com/pay/abc123?client_reference_id=order_456&metadata[user_id]=clerk_abc&metadata[plan]=pro
```

Semua parameter diteruskan ke webhook, sehingga Anda dapat merekonsiliasi pembayaran di backend Anda.

<Tip>
  Untuk integrasi perusahaan yang membutuhkan kontrol penuh, gunakan [Checkout Sessions API](/anyspend/api/checkout-sessions-api) untuk membuat sesi di sisi server dengan `customer_email`, `customer_name`, `metadata`, dan `client_reference_id`, kemudian arahkan menggunakan URL yang dikembalikan atau tambahkan `?session_id=` ke tautan pembayaran Anda.
</Tip>

## Daftar tautan pembayaran

<ParamField query="search" type="string">
  Cari tautan pembayaran berdasarkan nama atau deskripsi.
</ParamField>

<ParamField query="active" type="boolean">
  Filter berdasarkan status aktif. Lewati untuk mengembalikan semua tautan.
</ParamField>

<ParamField query="sort" type="string" default="created_at">
  Bidang pengurutan. Salah satu dari: `created_at`, `updated_at`, `name`, `uses`.
</ParamField>

<ParamField query="order" type="string" default="desc">
  Urutan pengurutan. Salah satu dari: `asc`, `desc`.
</ParamField>

<ParamField query="page" type="number" default={1}>
  Nomor halaman untuk paginasi.
</ParamField>

<ParamField query="limit" type="number" default={20}>
  Jumlah hasil per halaman (maks 100).
</ParamField>

<CodeGroup>
```bash cURL
curl -X GET "https://platform-api.anyspend.com/api/v1/payment-links?active=true&sort=created_at&order=desc&page=1&limit=20" \
  -H "Authorization: Bearer asp_xxx"
```

```typescript SDK
import { AnySpend } from "@b3dotfun/sdk";

const anyspend = new AnySpend(process.env.ANYSPEND_API_KEY!);

const { data, pagination } = await anyspend.paymentLinks.list({
  active: true,
  sort: "created_at",
  order: "desc",
  page: 1,
  limit: 20,
});
```
</CodeGroup>
