## Ikhtisar

AnySpend x402 mendukung pembayaran lintas berbagai jaringan blockchain termasuk rantai yang kompatibel dengan EVM dan Solana (SVM). Halaman ini menyediakan referensi lengkap dari jaringan yang didukung, token, dan alamat kontrak mereka.

## Kompatibilitas Token

<Note>
Untuk menggunakan token dengan X402, token tersebut harus mengimplementasikan **EIP-3009** (`transferWithAuthorization`) atau **EIP-2612** (`permit`) untuk tanda tangan tanpa gas.
</Note>

Daripada memelihara daftar token yang didukung secara statis, kami telah membangun alat untuk membantu Anda memverifikasi kompatibilitas token apa pun secara real-time.

## Pemeriksa Kompatibilitas Token X402

<Note>
**Alat Baru Tersedia**: Verifikasi apakah token ERC-20 apa pun kompatibel dengan pembayaran X402 sebelum integrasi.
</Note>

Agar token kompatibel dengan X402, token tersebut harus mengimplementasikan salah satu dari:
- **EIP-3009** (`transferWithAuthorization`) - Digunakan oleh USDC
- **EIP-2612** (`permit`) - Digunakan oleh USDT (di beberapa rantai) dan banyak token ERC-20 modern lainnya

### Menggunakan Pemeriksa Publik

Periksa token ERC-20 apa pun di jaringan yang didukung AnySpend:

**🌐 Antarmuka Web**: [https://anyspend.com/x402-tokens](https://anyspend.com/x402-tokens)

Cukup masukkan alamat token dan pilih jaringan untuk langsung memverifikasi kompatibilitas X402.

**🔗 API Langsung**: Anda juga dapat menanyakan API secara langsung:

```bash
curl https://tokens.anyspend.com/metadata/base/0xb3b32f9f8827d4634fe7d973fa1034ec9fddb3b3
```

Ini mengembalikan metadata token dan informasi kompatibilitas dalam format JSON.

### Menggunakan Paket NPM

Untuk pengembang yang mengintegrasikan pemeriksaan kompatibilitas token ke dalam aplikasi mereka:

**📦 Paket NPM**: [@b3dotfun/anyspend-x402-token-compat](https://www.npmjs.com/package/@b3dotfun/anyspend-x402-token-compat)

```bash
npm install @b3dotfun/anyspend-x402-token-compat
```

**Penggunaan:**
```typescript
import { TokenCompatClient } from '@b3dotfun/anyspend-x402-token-compat';

const client = new TokenCompatClient();

// Dapatkan metadata token dan kompatibilitas
const metadata = await client.getTokenMetadata('base', '0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913');
console.log(metadata.supportsEip2612); // true - mendukung permit
console.log(metadata.supportsEip3009); // true - mendukung transferWithAuth

// Pemeriksaan kompatibilitas cepat
const hasPermit = await client.supportsEip2612('base', tokenAddress);
```


### Cara Kerjanya

**Rute Lintas-Rantai yang Didukung:**
AnySpend mendukung perutean lintas-rantai antara jaringan EVM yang didukung, termasuk:
- Ethereum ↔ Base, Arbitrum, Optimism, Polygon, Avalanche
- Base ↔ Arbitrum, Optimism, Polygon
- L1 ↔ L2 bridging lintas semua rantai yang didukung
- Lintas-VM: Solana ↔ rantai EVM (eksperimental)

<Info>
Pembayaran lintas-rantai membutuhkan waktu lebih lama (2-5 menit) karena waktu bridging dan menimbulkan biaya tambahan (~0.5% total). Pembayaran dalam rantai yang sama direkomendasikan untuk pengalaman pengguna terbaik.
</Info>


<Note>
Dibangun di atas infrastruktur global Cloudflare dengan caching cerdas untuk hasil instan.
</Note>

Pemeriksa menggabungkan verifikasi on-chain real-time dengan database token yang diverifikasi yang terus bertambah:

- **⚡ Sangat Cepat**: Waktu respons tingkat milidetik melalui caching cerdas
- **📊 Bersumber dari Komunitas**: Database tumbuh secara otomatis seiring lebih banyak token yang diperiksa
- **🔄 Selalu Terbaru**: Verifikasi on-chain untuk token yang belum di-cache
- **🌍 Global**: Didukung oleh Cloudflare Workers untuk akses berlatensi rendah di seluruh dunia

### Integrasi di X402

Pemeriksa kompatibilitas token sudah terintegrasi ke dalam SDK AnySpend X402:

- Memvalidasi kompatibilitas token di depan sebelum pembayaran
- Mengembalikan kesalahan yang jelas dan dapat diambil tindakan untuk token yang tidak didukung
- Mencegah transaksi gagal karena token yang tidak kompatibel

**Cobalah secara langsung**: [https://x402-demo.anyspend.com/](https://x402-demo.anyspend.com/)

<Warning>
**Penting**: Selalu verifikasi kompatibilitas token sebelum mengintegrasikan token baru ke dalam alur pembayaran Anda. Token yang tidak kompatibel akan mengakibatkan gagalnya tanda tangan pembayaran.
</Warning>
