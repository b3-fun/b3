## ¿Qué es la Fase DEX?

La fase DEX es la etapa final del ciclo de vida de un token de Bondkit, donde la negociación transita de la curva de emisión automatizada a un pool completamente descentralizado de Uniswap v4. Esta fase comienza inmediatamente después de la migración y representa la descentralización completa.

```mermaid
stateDiagram-v2
    [*] --> BondingPhase: Token Creado
    BondingPhase --> MigrationReady: Objetivo Alcanzado
    MigrationReady --> DEXPhase: migrateToDex()
    DEXPhase --> [*]: Completamente Descentralizado
    
    note right of BondingPhase
        Precios controlados por contrato
        5% de comisiones de trading
        Solo compra/venta directa
        Acumulación de objetivo
    end note
    
    note right of DEXPhase
        Precios impulsados por el mercado
        Comisiones DEX estándar
        Funciones completas de trading
        Sin control administrativo
    end note
```

## Cómo Funciona la Migración

### El Proceso de Transición

Cuando se llama a `migrateToDex()` después de alcanzar el objetivo:

1. **Estado del Contrato**: El estado del token cambia a `TokenStatus.Dex`
2. **Pool de Uniswap v4**: Se crea un nuevo pool con la liquidez acumulada
3. **Inicialización del Precio**: El precio inicial del pool se calcula a partir del precio de salida de la curva de emisión
4. **Renuncia de Propiedad**: La propiedad del contrato se transfiere a la dirección cero
5. **Comercio Habilitado**: El comercio completo de DEX comienza inmediatamente

### Configuración del Pool

**Configuración Automática del Pool:**
```typescript
// El pool se crea con estos parámetros (desde la migración)
const poolConfig = {
  currency0: tradingToken < bondkitToken ? tradingToken : bondkitToken,
  currency1: tradingToken < bondkitToken ? bondkitToken : tradingToken,
  fee: 3000,        // Comisión estándar de 0.3%
  tickSpacing: 60,  // Espaciado estándar de ticks
  hooks: customHook // Hooks específicos de Bondkit
};
```

**Liquidez Inicial:**
- **Token de Trading**: Todos los fondos acumulados de la fase de emisión
- **Tokens de Bondkit**: Cantidad calculada para igualar el precio de salida
- **Tokens LP**: Divididos entre el receptor de comisiones y quemados/distribuidos

## Comercio en la Fase DEX

### Opciones de Comercio

**Comercio DEX Estándar:**
Los tokens de Bondkit en la fase DEX se pueden negociar usando cualquier interfaz compatible con Uniswap v4, agregador de DEX o plataforma de trading.

**SDK Helper (Opcional):**
El SDK proporciona `BondkitSwapService` para una integración simplificada si es necesario.

### Descubrimiento de Precios y Liquidez

**Precios Impulsados por el Mercado:**
- Sin más curva de emisión algorítmica
- Precio determinado por la oferta/demanda
- Dinámicas de libro de órdenes en tiempo real
- Sujeto a la volatilidad del mercado

**Fuentes de Liquidez:**
```typescript
// Liquidez inicial de la migración
const migrationData = {
  tradingTokenLiquidity: "10.0",    // ETH/B3 de la emisión
  bondkitTokenLiquidity: "500000",  // Cantidad de tokens calculada
  initialPrice: "0.00002",          // Precio por token
  totalValueLocked: "20.0"          // TVL combinado
};

// Liquidez adicional de:
// - LPs de la comunidad
// - Programas de incentivos  
// - Acumulación de comisiones de trading
```

## Diferencias Clave de la Fase de Emisión

### Mecánicas de Trading

| Aspecto | Fase de Emisión | Fase DEX |
|--------|--------------|-----------|
| **Precios** | Curva algorítmica | Oferta/demanda del mercado |
| **Liquidez** | Ilimitada (el contrato acuña) | Liquidez limitada del pool |
| **Comisiones** | 5% al receptor de comisiones | 0.3% a LPs + protocolo |
| **Tipos de Órdenes** | Solo compra/venta | Funciones completas de DEX |
| **Impacto de Precio** | Curva predecible | Depende de la profundidad de liquidez |
| **MEV** | No aplicable | Sujeto a MEV |

### Cambios en el Contrato Inteligente

**Transición de Estado:**
```typescript
// Verificar la fase actual
const status = await token.getStatus();

if (status === TokenStatus.Dex) {
  // Usar métodos de trading de DEX
  const swapService = new BondkitSwapService(tokenAddress);
  // ... código de trading de DEX
} else if (status === TokenStatus.Bonding) {
  // Usar métodos de curva de emisión
  await token.buy(amount, minTokensOut);
}
```

**Disponibilidad de Métodos:**
- ✅ `token.buy()` / `token.sell()` - **Deshabilitado** (revertirá)
- ✅ Trading de DEX vía `BondkitSwapService`
- ✅ La funcionalidad ERC20 estándar continúa
- ✅ Monitoreo de eventos para trades de DEX

## Análisis del Impacto de la Migración

### Para los Tenedores de Tokens

**Efectos Inmediatos:**
- ✅ Los tokens siguen siendo completamente transferibles
- ✅ Pueden comerciar en el mercado abierto
- ✅ Sin períodos de bloqueo
- ⚠️ El precio ahora está sujeto a la volatilidad del mercado
- ⚠️ No hay liquidez garantizada (depende de la profundidad del pool)

**Beneficios a Largo Plazo:**
- 🚀 Acceso a un ecosistema de trading más grande
- 🚀 Integración con protocolos DeFi
- 🚀 Composabilidad con otros tokens
- 🚀 Verdadera descentralización lograda

### Para Proyectos

**Cambios Operacionales:**
```typescript
// Pre-migración: Control directo
await token.buy(amount, minOut);    // ✅ Funciona
await token.migrateToDex();         // Acción única

// Post-migración: Basado en el mercado
await token.buy(amount, minOut);    // ❌ Revierte
await swapService.executeSwap(...); // ✅ Usar esto en su lugar
```

**Nuevas Responsabilidades:**
- **Construcción de Comunidad**: Impulsar el volumen de trading orgánico
- **Incentivos de Liquidez**: Fomentar la participación de LP  
- **Marketing**: Promover la utilidad y adopción del token
- **Integración**: Listar en agregadores de DEX y plataformas

## Preguntas Comunes

<AccordionGroup>
  <Accordion title="¿Pueden los tokens volver a la fase de emisión?">
    **No** - La migración es permanente e irreversible. Una vez en la fase DEX, el token siempre se negociará en Uniswap v4.
  </Accordion>
  
  <Accordion title="¿Qué sucede con la liquidez no utilizada de la curva de emisión?">
    Todos los tokens de trading acumulados de la fase de emisión se convierten en la liquidez inicial en el pool de Uniswap v4, emparejados con tokens de Bondkit calculados al precio de salida.
  </Accordion>
  
  <Accordion title="¿Hay alguna restricción en la fase DEX?">
    Sin restricciones - los tokens se convierten en ERC20s estándar que se negocian en Uniswap v4. Se habilita la composabilidad completa de DeFi.
  </Accordion>
  
  <Accordion title="¿Cómo son diferentes las comisiones de trading?">
    La fase DEX utiliza comisiones estándar de Uniswap v4 (0.3%) que van a los proveedores de liquidez en lugar de la comisión de la fase de emisión del 5% al receptor de comisiones.
  </Accordion>
</AccordionGroup>
