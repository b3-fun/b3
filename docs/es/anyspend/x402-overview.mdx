## ¿Qué es AnySpend x402?

AnySpend x402 es la implementación de B3 del **protocolo de pago x402** que permite pagos automáticos e instantáneos de criptomonedas directamente sobre HTTP. A diferencia de los facilitadores tradicionales de x402 que solo admiten USDC, **AnySpend te permite pagar con cualquier token** mientras que los servidores de recursos reciben cualquier token que prefieran.

El protocolo x402 moderniza el código de estado HTTP `402 Payment Required`, convirtiéndolo en una solución práctica para micropagos de máquina a máquina, APIs con muro de pago y interacciones de agentes autónomos.

## Para Compradores (Pagadores)

Paga por recursos con muro de pago usando cualquier criptomoneda que poseas:

- **Paga con cualquier token**: Usa ETH, DAI, o cualquier token ERC-20 - no solo USDC
- **Protocolo estándar**: Funciona con cualquier servicio habilitado para x402 usando HTTP estándar
- **Sin cambios manuales**: AnySpend convierte automáticamente tu token a lo que el vendedor necesita
- **Integración simple**: Un SDK funciona con todos los servicios x402

**Casos de uso:**
- Acceder a APIs premium y servicios de agentes de IA
- Pagar por recursos computacionales y fuentes de datos
- Habilitar pagos autónomos de agente a agente
- Consumir contenido con muro de pago programáticamente

## Para Vendedores (Servidores de Recursos)

Acepta pagos en USDC mientras tus clientes pagan con su token preferido:

- **Recibe USDC**: Se paga en USDC independientemente del token que use el cliente
- **Middleware de Express**: Agrega pagos a tu API con solo unas pocas líneas de código
- **Sin infraestructura de blockchain**: AnySpend maneja toda la complejidad en cadena
- **Liquidación instantánea**: Los pagos se verifican y liquidan automáticamente

**Casos de uso:**
- Monetizar APIs premium y servicios de datos
- Construir flujos de trabajo de agentes de IA pagados
- Crear contenido y herramientas con muro de pago
- Aceptar pagos en cripto sin conocimientos de criptomonedas

## Cómo Funciona

<Steps>
<Step title="El Cliente Solicita el Recurso">
La aplicación del usuario realiza una solicitud HTTP estándar a tu API
</Step>

<Step title="Pago Requerido">
Tu API devuelve `402 Payment Required` con los detalles del pago
</Step>

<Step title="Preferencia de Token">
El cliente especifica su token de pago preferido (ETH, DAI, etc.)
</Step>

<Step title="Cotización de AnySpend">
AnySpend calcula la cantidad equivalente en el token elegido por el cliente
</Step>

<Step title="El Cliente Firma el Pago">
El usuario firma una autorización criptográfica para su token
</Step>

<Step title="Liquidación Automática">
AnySpend intercambia el token por USDC y lo liquida a tu cartera
</Step>

<Step title="Recurso Entregado">
Tu API recibe la confirmación del pago y devuelve el recurso solicitado
</Step>
</Steps>

## Diferenciadores Clave

### Soporte Multi-Token

**x402 estándar**: Los servidores de recursos y los clientes deben usar ambos USDC

**AnySpend x402**: Los clientes pagan con **cualquier token** que posean. AnySpend maneja la conversión.

```
El cliente paga 0.0003 ETH → AnySpend lo cambia por 1 USDC → El vendedor recibe 1 USDC
```

### Sin Infraestructura Requerida

Los servidores de recursos no necesitan:
- Ejecutar nodos de blockchain
- Gestionar carteras o claves privadas
- Manejar intercambios de tokens o puentes
- Monitorear el estado de las transacciones

El facilitador de AnySpend maneja todas las operaciones de blockchain y proporciona puntos finales HTTP simples para la verificación y liquidación.

### Cumplimiento con el Protocolo Estándar

Los clientes que usan AnySpend tienen **cero conciencia** de que están usando un facilitador especial. Todo el flujo utiliza mensajes de protocolo x402 estándar - la magia multi-token ocurre de manera transparente en el lado del servidor de recursos a través de middleware.

Todos los pagos x402 se procesan a través del **Facilitador de AnySpend** - una infraestructura de pago de grado de producción que maneja:

- ✅ Soporte multi-token a través de más de 19 redes blockchain
- ✅ Intercambios automáticos de tokens y enrutamiento entre cadenas
- ✅ Pagos sin gas para los usuarios
- ✅ Implementación conforme a los estándares de x402
- ✅ Reemplazo directo para cualquier facilitador de x402
