## Reglas de Validación de IA

El punto final de validación de IA puntúa tu idea de artículo y la aprueba tal cual, sugiere mejoras o la rechaza.

<Tabs>
  <Tab title="✅ Puntuación ≥80">
    **¡Perfecto!** Tu entrada es aprobada tal cual.
    
    Tu título o tema proporcionado es excelente y se ajusta al enfoque del sitio. No se necesitan cambios - lo enviaremos exactamente como lo escribiste.
    
    **Ejemplo:** "Cómo Usar Base Pay para Creadores en Tu DApp"
  </Tab>
  <Tab title="⚠️ Puntuación 70-79">
    **Bien con Sugerencias!** La IA proporciona una versión mejorada.
    
    Tu contenido es relevante pero podría ser optimizado. La IA genera un mejor título y descripción manteniendo tu idea central.
    
    **Entrada de Ejemplo:** "base pay para creadores"
    
    **Salida de IA:** "Integrando Base Pay: Una Guía Completa para Creadores de Contenido"
  </Tab>
  <Tab title="❌ Puntuación <70">
    **Rechazado.** El contenido no se ajusta al enfoque del sitio.
    
    Tu tema está fuera de tema para este sitio. Prueba un tema diferente o elige un sitio diferente que se ajuste mejor a tu contenido.
    
    **Ejemplo:** Enviar una receta de cocina a un sitio de noticias de cripto
  </Tab>
</Tabs>

### Validar Idea de Artículo (Gratis)

```bash
POST https://x402.anyspend.com/articles/validate
```

Punto final gratuito para validar tu idea de artículo. La IA genera un título, descripción y puntuación de aprobación.

**Cuerpo de la Solicitud:**

<CodeGroup>
```json Request
{
  "siteDomain": "baseradar.com",
  "userInput": "How to use Base Pay for creators"
}
```

```json Success Response (200) - Approved
{
  "success": true,
  "data": {
    "approved": true,
    "score": 85,
    "reasoning": "Buen tema para sitio enfocado en Base. La IA optimizó el título para un mejor compromiso.",
    "suggestedTitle": "Integrando Base Pay: Una Guía Completa para Creadores de Contenido",
    "suggestedDescription": "Aprende cómo implementar Base Pay en tu plataforma de creadores con ejemplos paso a paso y mejores prácticas.",
    "estimatedPrice": "50000000",
    "userProvidedTitle": false,
    "message": "Contenido aprobado con sugerencias de IA"
  }
}
```

```json Success Response (200) - Perfect Score
{
  "success": true,
  "data": {
    "approved": true,
    "score": 95,
    "reasoning": "¡Título excelente! Claro, específico y muy relevante para la audiencia del sitio.",
    "suggestedTitle": "Cómo Usar Base Pay para Creadores en Tu DApp",
    "suggestedDescription": "Guía completa de implementación para Base Pay",
    "estimatedPrice": "50000000",
```

```json Rejection Response (200) - Score Too Low
  "data": {

    "score": 45,
```

**Parámetros:**
- `siteDomain` (obligatorio) - Dominio del sitio objetivo (por ejemplo, "baseradar.com")
- `userInput` (obligatorio) - URL de tweet, idea de tema o título completo (10-200 caracteres)

**Opciones de Entrada:**

### Enviar Solicitud de Artículo (Pago con x402)

```bash
POST https://x402.anyspend.com/articles/buy
```

Envía una solicitud de artículo con pago x402. Debes validar tu contenido primero y recibir aprobación.

**Cuerpo de la Solicitud:**

<CodeGroup>
```json Request
{
  "buyerAddress": "0x742d35Cc6...",
  "siteDomain": "baseradar.com",
  "title": "Integrando Base Pay: Una Guía Completa para Creadores de Contenido",
  "description": "Aprende cómo implementar Base Pay en tu plataforma de creadores con ejemplos paso a paso y mejores prácticas."
}
```

```json Success Response (200)
{
  "success": true,
  "data": {
    "requestId": "req_789...",
    "productSpecificId": "req_789...",
    "totalPrice": "50000000",
    "status": "pending"
  }
}
```

```json Error Response (400)
{
  "success": false,
  "error": "Se requiere validación de contenido antes de la presentación"
}
```
</CodeGroup>

**Parámetros:**
- `buyerAddress` (obligatorio) - Tu dirección de cartera
- `siteDomain` (obligatorio) - Dominio del sitio objetivo
- `title` (obligatorio) - Título del artículo (de la validación o el tuyo si la puntuación ≥90)
- `description` (obligatorio) - Descripción del artículo (de la validación o la tuya)

**Ejemplo en JavaScript/TypeScript:**

```typescript
import { wrapFetchWithPayment } from '@b3dotfun/anyspend-x402-fetch';
import { createWalletClient, custom, publicActions } from 'viem';
import { base } from 'viem/chains';

// Paso 1: Valida tu contenido (GRATIS)
const validateResponse = await fetch('https://x402.anyspend.com/articles/validate', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    siteDomain: 'baseradar.com',
    userInput: 'How to use Base Pay for creators'
  })
});

const validation = await validateResponse.json();

if (!validation.data.approved) {
  console.error('Contenido rechazado:', validation.data.reasoning);
  process.exit(1);
}

console.log('✅ ¡Aprobado! Puntuación:', validation.data.score);
console.log('Título:', validation.data.suggestedTitle);

// Paso 2: Enviar con pago (x402)
const walletClient = createWalletClient({
  account: '0x742d35Cc6...',
  chain: base,
  transport: custom(window.ethereum),
}).extend(publicActions);

const fetchWithPayment = wrapFetchWithPayment(
  fetch,
  walletClient,
  BigInt(1000 * 10 ** 6) // Máximo 1000 USDC
);

const response = await fetchWithPayment(
  'https://x402.anyspend.com/articles/buy',
  {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      buyerAddress: '0x742d35Cc6...',
      siteDomain: 'baseradar.com',
      title: validation.data.suggestedTitle,
      description: validation.data.suggestedDescription
    })
  }
);

const result = await response.json();
console.log('🎉 ¡Artículo enviado! ID de solicitud:', result.data.requestId);
```

<Note>
  Consulta la [documentación del SDK de x402](/anyspend/x402-overview) para obtener más detalles sobre el uso de `wrapFetchWithPayment`.
</Note>

## Consejos de Validación de IA

<AccordionGroup>
  <Accordion title="Escribe Títulos Claros y Específicos">
    **Bien:** "Cómo Construir un DEX en Base con Uniswap V3"
    
    **Mal:** "Cosas de DeFi"
    
    Los títulos específicos obtienen puntuaciones más altas y tienen más probabilidades de ser aprobados.
  </Accordion>
  
  <Accordion title="Coincide con el Enfoque del Sitio">
    Verifica las palabras clave y descripción del sitio antes de enviar. Un artículo enfocado en Base no puntuará bien en un sitio solo de Solana.
  </Accordion>
  
  <Accordion title="Usa Oraciones Completas para los Títulos">
    **Bien:** "Cómo Usar Base Pay para Creadores en Tu DApp"
    
    **Mal:** "base pay creadores"
    
    Los títulos completos (10-200 caracteres) obtienen puntuaciones más altas.
  </Accordion>
  
  <Accordion title="Aprovecha las URLs de Tweets">
    Si tienes un tweet viral sobre tu tema, ¡usa la URL! La IA extrae el contexto y genera un título relevante.
  </Accordion>
</AccordionGroup>

## Problemas Comunes

<AccordionGroup>
  <Accordion title="'Contenido rechazado' con puntuación baja">
    Tu tema no se ajusta al enfoque del sitio. Opciones:
    - Elige un sitio diferente que coincida mejor con tu tema
    - Ajusta tu tema para alinearlo con las palabras clave del sitio
    - Proporciona un título más específico y detallado
  </Accordion>
  
  <Accordion title="Error 'Se requiere validación de contenido'">
    Debes llamar primero a `/articles/validate` y obtener aprobación antes de llamar a `/articles/buy`.
  </Accordion>
  
  <Accordion title="Error 'Saldo insuficiente'">
    Necesitas USDC en la red Base. Asegúrate de que:
    - Estás conectado a Base (Chain ID: 8453)
    - Tienes suficiente USDC para el precio del artículo
  </Accordion>
  
  <Accordion title="¿Qué sucede después de enviar?">
    1. Tu solicitud va al editor para revisión
    2. El editor evalúa tu propuesta (típicamente 1-7 días)
    3. Si se acepta, escribirán/publicarán el artículo
    4. Recibirás la URL publicada una vez que esté
