Sesiones de pago al estilo Stripe para AnySpend. Las sesiones están desacopladas de los pedidos: primero se crea una sesión y luego se crea un pedido cuando el usuario está listo para pagar.

## Flujo

```
1. El comerciante crea la sesión        POST /checkout-sessions
                                        <- { id, status: "open" }

2. El usuario elige el método de pago   POST /orders { checkoutSessionId }
                                        <- { id, globalAddress, oneClickBuyUrl }

3. El usuario paga                      Cripto: enviar a globalAddress
                                        Onramp: redirigir a oneClickBuyUrl

4. El comerciante verifica la finalización   GET /checkout-sessions/:id
                                             <- { status: "complete", order_id }
```

### ¿Por qué Desacoplado?

La creación de la sesión es instantánea (solo DB, sin llamadas externas). El pedido se crea por separado cuando el usuario se compromete con un método de pago. Esto significa:

- No es necesario conocer el método de pago al crear la sesión
- Una página de pago alojada puede permitir a los usuarios elegir cómo pagar
- La creación de la sesión nunca falla debido a errores de API externas

## Ciclo de Vida del Estado de la Sesión

```
open --> processing --> complete
  |
  └--> expired
```

| Estado      | Cuando |
|-------------|--------|
| `open`      | Creada, esperando pedido/pago |
| `processing`| Pago recibido, pedido en ejecución |
| `complete`  | Pedido ejecutado con éxito |
| `expired`   | TTL expirado, pago fallido o expirado manualmente |

## API

### `POST /checkout-sessions` — Crear Sesión

Crea una sesión ligera. Sin pedido, sin llamadas a API externas.

```json
{
  "success_url": "https://comerciante.com/success?session_id={SESSION_ID}",
  "cancel_url": "https://comerciante.com/cancel",
  "metadata": { "sku": "widget-1" },
  "client_reference_id": "pedido-comerciante-456",
  "expires_in": 1800
}
```

Todos los campos son opcionales. La configuración de pago (monto, tokens, cadenas) reside en el pedido, no en la sesión.

### `POST /orders` — Crear Pedido con Enlace a Sesión

Pasa `checkoutSessionId` en la solicitud estándar de creación de pedido para enlazar el pedido a una sesión.

```json
{
  "recipientAddress": "0x...",
  "srcChain": 8453,
  "dstChain": 8453,
  "srcTokenAddress": "0x...",
  "dstTokenAddress": "0x...",
  "srcAmount": "1000000",
  "type": "swap",
  "payload": { "expectedDstAmount": "1000000" },
  "checkoutSessionId": "550e8400-..."
}
```

**Validación:**
- La sesión debe existir (`400` si no se encuentra)
- La sesión debe estar `open` (`400` si está expirada/procesando/completa)
- La sesión no debe tener ya un pedido (`409 Conflict`)

### `GET /checkout-sessions/:id` — Recuperar Sesión

Devuelve el estado actual de la sesión. El estado se sincroniza con el pedido subyacente en cada recuperación.

| Parámetro de Consulta | Descripción |
|-----------------------|-------------|
| `include=order`       | Incorpora el objeto de pedido completo con transacciones |

### `POST /checkout-sessions/:id/expire` — Expirar Manualmente

Solo funciona en sesiones con estado `open`.

## Plantillas de URL de Redirección

Usa variables de plantilla en `success_url` y `cancel_url`:

| Variable       | Reemplazado con |
|----------------|-----------------|
| `{SESSION_ID}` | El UUID de la sesión de pago |
| `{ORDER_ID}`   | Mismo valor (alias) |

Si no hay una variable de plantilla presente, `?sessionId=<uuid>` se añade automáticamente.

## Integración SDK

### Métodos del Servicio

```typescript
// Crear una sesión de pago
const session = await anyspend.createCheckoutSession({
  success_url: "https://misitio.com/success/{SESSION_ID}",
  metadata: { sku: "widget-1" },
});

// Recuperar el estado de la sesión
const session = await anyspend.getCheckoutSession(sessionId);
```

### React Hooks

#### `useCreateCheckoutSession`

Hook de mutación para crear sesiones.

```tsx
import { useCreateCheckoutSession } from "@b3dotfun/sdk/anyspend";

const { mutate: createSession, data, isPending } = useCreateCheckoutSession();
```

#### `useCheckoutSession`

Hook de consulta con auto-sondeo. Detiene el sondeo cuando el estado alcanza `complete` o `expired`.

```tsx
import { useCheckoutSession } from "@b3dotfun/sdk/anyspend";

const { data: session, isLoading } = useCheckoutSession(sessionId);
```

### Prop `checkoutSession` del Componente

Los componentes `<AnySpend>`, `<AnySpendCustom>`, y `<AnySpendCustomExactIn>` aceptan una prop `checkoutSession` opcional:

```tsx
<AnySpend
  defaultActiveTab="fiat"
  destinationTokenAddress="0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913"
  destinationTokenChainId={8453}
  recipientAddress="0x..."
  checkoutSession={{
    success_url: "https://mitienda.com/success?session={SESSION_ID}",
    cancel_url: "https://mitienda.com/cancel",
    metadata: { sku: "widget-1" },
  }}
/>
```

Cuando se establece la prop `checkoutSession`, el componente crea automáticamente una sesión antes de crear el pedido y utiliza el `success_url` de la sesión para redirecciones. Sin la prop, los flujos existentes no cambian.

## Ejemplos

### Pago con Cripto

```typescript
// 1. Crear sesión
const session = await fetch("/checkout-sessions", {
  method: "POST",
  body: JSON.stringify({
    success_url: "https://misitio.com/success/{SESSION_ID}",
    metadata: { sku: "widget-1" },
  }),
}).then(r => r.json());

// 2. Crear pedido vinculado a la sesión
const order = await fetch("/orders", {
  method: "POST",
  body: JSON.stringify({
    recipientAddress: "0x...",
    srcChain: 8453,
    dstChain: 8453,
    srcTokenAddress: "0x833589fcd6edb6e08f4c7c32d4f71b54bda02913",
    dstTokenAddress: "0x833589fcd6edb6e08f4c7c32d4f71b54bda02913",
    srcAmount: "1000000",
    type: "swap",
    payload: { expectedDstAmount: "1000000" },
    checkoutSessionId: session.data.id,
  }),
}).then(r => r.json());

// 3. El usuario envía cripto a order.data.globalAddress

// 4. Sondear la sesión hasta completar
const poll = setInterval(async () => {
  const s = await fetch(`/checkout-sessions/${session.data.id}`).then(r => r.json());
  if (s.data.status === "complete") {
    clearInterval(poll);
    // redirigir a success_url o mostrar confirmación
  }
}, 3000);
```

### Pago Onramp (Coinbase/Stripe)

```typescript
// Los pasos 1-2 son los mismos que arriba, pero incluyen la configuración de onramp en la creación del pedido:
const order = await fetch("/orders", {
  method: "POST",
  body: JSON.stringify({
    // ... mismos campos del pedido ...
    checkoutSessionId: session.data.id,
    onramp: {
      vendor: "coinbase",
      payment_method: "card",
      country: "US",
    },
  }),
}).then(r => r.json());

// Redirigir al usuario a la página de pago del proveedor
window.location.href = order.data.oneClickBuyUrl;

// Después de que el proveedor redirija de vuelta, sondear GET /checkout-sessions/:id para completar
