## Visión general

AnySpend x402 soporta pagos a través de múltiples redes blockchain incluyendo cadenas compatibles con EVM y Solana (SVM). Esta página proporciona la referencia completa de redes soportadas, tokens y sus direcciones de contrato.

AnySpend x402 soporta las siguientes cadenas principales EVM:

<Info>
El soporte de Solana habilita pagos cruzados de VM - los usuarios pueden pagar con SOL o tokens SPL mientras que los servidores de recursos reciben stablecoins en cadenas EVM.
</Info>

## Compatibilidad de tokens

<Note>
Para usar un token con X402, debe implementar **EIP-3009** (`transferWithAuthorization`) o **EIP-2612** (`permit`) para firmas sin gas.
</Note>

En lugar de mantener una lista estática de tokens soportados, hemos construido herramientas para ayudarte a verificar la compatibilidad de cualquier token en tiempo real.

## Verificador de compatibilidad de tokens X402

<Note>
**Nueva Herramienta Disponible**: Verifica si cualquier token ERC-20 es compatible con pagos X402 antes de la integración.
</Note>

Para que un token sea compatible con X402, debe implementar:
- **EIP-3009** (`transferWithAuthorization`) - Usado por USDC
- **EIP-2612** (`permit`) - Usado por USDT (en algunas cadenas) y muchos otros tokens ERC-20 modernos

### Usando el verificador público

Verifica cualquier token ERC-20 en cadenas soportadas por AnySpend:

**Interfaz web**: [anyspend.com/x402-tokens](https://anyspend.com/x402-tokens)

Simplemente ingresa una dirección de token y selecciona la red para verificar instantáneamente la compatibilidad con X402.

**API directa**: También puedes consultar la API directamente:

```bash
curl https://tokens.anyspend.com/metadata/base/0xb3b32f9f8827d4634fe7d973fa1034ec9fddb3b3
```

Esto devuelve los metadatos del token y la información de compatibilidad en formato JSON.

### Usando el paquete npm

Para desarrolladores que integran verificaciones de compatibilidad de tokens en sus aplicaciones:

**paquete npm**: [@b3dotfun/anyspend-x402-token-compat](https://www.npmjs.com/package/@b3dotfun/anyspend-x402-token-compat)

```bash
npm install @b3dotfun/anyspend-x402-token-compat
```

**Uso:**
```typescript
import { TokenCompatClient } from '@b3dotfun/anyspend-x402-token-compat';

const client = new TokenCompatClient();

// Obtener metadatos del token y compatibilidad
const metadata = await client.getTokenMetadata('base', '0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913');
console.log(metadata.supportsEip2612); // true - soporta permit
console.log(metadata.supportsEip3009); // true - soporta transferWithAuth

// Verificación rápida de compatibilidad
const hasPermit = await client.supportsEip2612('base', tokenAddress);
```


### Cómo funciona

**Rutas Cruzadas de Cadena Soportadas:**
AnySpend soporta enrutamiento cruzado de cadena entre cualquiera de las redes EVM soportadas, incluyendo:
- Ethereum ↔ Base, Arbitrum, Optimism, Polygon, Avalanche
- Base ↔ Arbitrum, Optimism, Polygon
- L1 ↔ L2 puenteando a través de todas las cadenas soportadas
- Cruzado de VM: Solana ↔ cadenas EVM (experimental)

<Info>
Los pagos entre cadenas tardan más (2-5 minutos) debido al tiempo de puenteo e incurren en tarifas adicionales (~0.5% total). Se recomiendan pagos en la misma cadena para la mejor experiencia de usuario.
</Info>


<Note>
Construido sobre la infraestructura global de Cloudflare con caché inteligente para resultados instantáneos.
</Note>

El verificador combina la verificación en cadena en tiempo real con una base de datos en crecimiento de tokens verificados:

- Tiempos de respuesta a nivel de milisegundos a través de caché inteligente
- La base de datos crece automáticamente a medida que se verifican más tokens
- Verificación en cadena para tokens no en caché
- Funciona en Cloudflare Workers para acceso de baja latencia en todo el mundo

### Integración en x402

El verificador de compatibilidad de tokens ya está integrado en el SDK de AnySpend X402:

- Valida la compatibilidad del token de antemano antes del pago
- Devuelve errores claros y accionables para tokens no soportados
- Previene transacciones fallidas debido a tokens incompatibles

**Pruébalo en vivo**: [https://x402-demo.anyspend.com/](https://x402-demo.anyspend.com/)

<Warning>
**Importante**: Siempre verifica la compatibilidad del token antes de integrar un nuevo token en tu flujo de pago. Los tokens incompatibles resultarán en firmas de pago fallidas.
</Warning>
