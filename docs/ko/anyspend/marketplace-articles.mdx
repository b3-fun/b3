## AI 검증 규칙

AI 검증 엔드포인트는 귀하의 기사 아이디어를 평가하고 그대로 승인하거나 개선을 제안하거나 거절합니다.

<Tabs>
  <Tab title="✅ 점수 ≥80">
    **완벽!** 귀하의 입력이 그대로 승인되었습니다.
    
    제공된 제목 또는 주제가 사이트의 초점과 훌륭하게 일치합니다. 변경할 필요가 없습니다 - 귀하가 작성한 그대로 제출하겠습니다.
    
    **예시:** "귀하의 DApp에서 크리에이터를 위한 Base Pay 사용 방법"
  </Tab>
  <Tab title="⚠️ 점수 70-79">
    **좋음, 제안과 함께!** AI가 개선된 버전을 제공합니다.
    
    귀하의 콘텐츠는 관련이 있지만 최적화될 수 있습니다. AI가 귀하의 핵심 아이디어를 유지하면서 더 나은 제목과 설명을 생성합니다.
    
    **예시 입력:** "크리에이터를 위한 base pay"
    
    **AI 출력:** "콘텐츠 크리에이터를 위한 Base Pay 통합: 완벽 가이드"
  </Tab>
  <Tab title="❌ 점수 <70">
    **거절.** 콘텐츠가 사이트의 초점에 맞지 않습니다.
    
    귀하의 주제가 이 사이트의 주제와 맞지 않습니다. 다른 주제를 시도하거나 콘텐츠와 더 잘 맞는 다른 사이트를 선택하세요.
    
    **예시:** 요리 레시피를 암호화폐 뉴스 사이트에 제출하기
  </Tab>

## 가격

기사 제출 가격은 해당 사이트의 트래픽과 권위에 따라 다릅니다.

**일반적인 범위:**
- 소규모/신생 사이트: 제출당 $3-50 USDC
- 중간 규모 사이트: 제출당 $50-200 USDC
- 프리미엄 사이트: 제출당 $200-500 USDC

<Note>
  가격은 개별 출판사가 설정합니다. 제출하기 전에 사이트 세부 정보를 확인하세요.
</Note>

### 기사 요청 제출 (x402 결제)

```bash
POST https://x402.anyspend.com/articles/buy
```

x402 결제로 기사 요청을 제출합니다. 먼저 귀하의 콘텐츠를 검증하고 승인을 받아야 합니다.

**요청 본문:**

<CodeGroup>
```json Request
{
  "buyerAddress": "0x742d35Cc6...",
  "siteDomain": "baseradar.com",
  "title": "콘텐츠 크리에이터를 위한 Base Pay 통합: 완벽 가이드",
  "description": "단계별 예시와 모범 사례를 통해 귀하의 크리에이터 플랫폼에서 Base Pay를 구현하는 방법을 배웁니다."
}
```

```json Success Response (200)
{
  "success": true,
  "data": {
    "requestId": "req_789...",
    "productSpecificId": "req_789...",
    "totalPrice": "50000000",
    "status": "pending"
  }
}
```

```json Error Response (400)
{
  "success": false,
  "error": "제출 전에 '/articles/validate'를 호출하고 승인을 받아야 합니다"
}
```
</CodeGroup>

**매개변수:**
- `buyerAddress` (필수) - 귀하의 지갑 주소
- `siteDomain` (필수) - 대상 사이트 도메인
- `title` (필수) - 기사 제목 (검증에서 얻은 것 또는 점수 ≥90인 경우 귀하의 것)
- `description` (필수) - 기사 설명 (검증에서 얻은 것 또는 귀하의 것)

**JavaScript/TypeScript 예시:**

```typescript
import { wrapFetchWithPayment } from '@b3dotfun/anyspend-x402-fetch';
import { createWalletClient, custom, publicActions } from 'viem';
import { base } from 'viem/chains';

// 1단계: 콘텐츠 검증 (무료)
const validateResponse = await fetch('https://x402.anyspend.com/articles/validate', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    siteDomain: 'baseradar.com',
    userInput: '크리에이터를 위한 Base Pay 사용 방법'
  })
});

const validation = await validateResponse.json();

if (!validation.data.approved) {
  console.error('콘텐츠 거절:', validation.data.reasoning);
  process.exit(1);
}

console.log('✅ 승인됨! 점수:', validation.data.score);
console.log('제목:', validation.data.suggestedTitle);

// 2단계: 결제와 함께 제출 (x402)
const walletClient = createWalletClient({
  account: '0x742d35Cc6...',
  chain: base,
  transport: custom(window.ethereum),
}).extend(publicActions);

const fetchWithPayment = wrapFetchWithPayment(
  fetch,
  walletClient,
  BigInt(1000 * 10 ** 6) // 최대 1000 USDC
);

const response = await fetchWithPayment(
  'https://x402.anyspend.com/articles/buy',
  {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      buyerAddress: '0x742d35Cc6...',
      siteDomain: 'baseradar.com',
      title: validation.data.suggestedTitle,
      description: validation.data.suggestedDescription
    })
  }
);

const result = await response.json();
console.log('🎉 기사 제출됨! 요청 ID:', result.data.requestId);
```

<Note>
  `wrapFetchWithPayment` 사용에 대한 자세한 내용은 [x402 SDK 문서](/anyspend/x402-overview)를 참조하세요.
</Note>

**상태 값:**
- `pending` - 출판사에 제출됨, 검토 대기 중
- `accepted` - 출판사가 수락함, 기사 제작 중
- `published` - 기사가 게시됨 (`publishedUrl` 포함)
- `rejected` - 출판사가 거절함 (드물게, 보통 사전 검증됨)

<Info>
  API 응답의 가격은 USDC wei(6자리 소수점)입니다. USDC 금액을 얻으려면 1,000,000으로 나누세요.
</Info>
