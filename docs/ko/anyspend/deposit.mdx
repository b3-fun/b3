## 무엇을 하는가

AnySpend 입금 페이지는 사용자가 특정 수신자 지갑 주소에 토큰을 입금할 수 있는 간소화된 인터페이스를 제공합니다. 이 기능은 교차 체인 입금을 지원하여 사용자가 지원되는 모든 체인의 토큰으로 결제할 수 있고 수신자는 원하는 목적지 체인과 토큰으로 자금을 받을 수 있습니다.

이 기능은 사용자로부터 토큰 입금을 받아야 하는 dApp, 봇, 서비스와 통합하기 위해 설계되었습니다.

## 작동 방식

1. **딥 링크 생성**\
   귀하의 애플리케이션이 수신자 주소, 목적지 체인, 토큰과 함께 입금 URL을 생성합니다.
2. **사용자 지갑 연결**\
   사용자가 자신의 지갑(MetaMask, Coinbase, Rainbow, Rabby, Trust Wallet 등)을 연결합니다.
3. **토큰 선택**\
   사용자가 사용 가능한 잔액에서 결제할 토큰을 선택합니다.
4. **교차 체인 실행**\
   AnySpend이 교차 체인 라우팅 및 변환을 처리하여 지정된 토큰을 수신자에게 전달합니다.
5. **확인 및 리디렉션**\
   완료 후 사용자는 귀하의 애플리케이션으로 다시 리디렉션될 수 있습니다.

## URL 파라미터

입금 페이지는 다음과 같은 URL 파라미터를 받습니다:

| 파라미터 | 필수 | 설명 |
|-----------|----------|-------------|
| `recipientAddress` | 예 | 입금을 받을 지갑 주소 (유효한 EVM 주소) |
| `toChainId` | 예* | 목적지 체인 ID (예: Base의 경우 8453) |
| `toCurrency` | 예* | 목적지 체인의 토큰 계약 주소 |
| `amount` | 아니요 | wei/최소 단위로 고정된 금액. 제공된 경우 사용자는 금액을 변경할 수 없습니다 |
| `redirect_url` | 아니요 | 완료 후 리디렉션할 HTTPS URL |
| `redirect_label` | 아니요 | 리디렉션 버튼의 사용자 정의 레이블 |
| `partner` | 아니요 | 사용자 정의 구성을 위한 파트너 ID |

\* Hyperliquid 체인에서는 USDC가 기본 토큰으로 설정되어 있어 필요하지 않습니다.

## 예시 URL

**Base USDC로 기본 입금:**
```
/deposit?recipientAddress=0x1234...&toChainId=8453&toCurrency=0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913
```

**고정된 금액 입금:**
```
/deposit?recipientAddress=0x1234...&toChainId=8453&toCurrency=0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913&amount=1000000
```

**리디렉션 포함:**
```
/deposit?recipientAddress=0x1234...&toChainId=8453&toCurrency=0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913&redirect_url=https://yourapp.com/success&redirect_label=앱으로 돌아가기
```

## 지원되는 지갑

입금 페이지는 기본적으로 다음 지갑을 지원합니다:

- MetaMask
- Coinbase Wallet
- Rainbow
- Rabby
- Trust Wallet

이전에 지갑을 연결한 사용자의 경우 자동 연결이 활성화됩니다.

## 특별 체인 지원

### Hyperliquid

입금 페이지는 Hyperliquid 체인에 대한 특별 지원을 포함합니다:

- USDC는 특별한 34자리 주소 형식(`0x00000000000000000000000000000000`)을 사용합니다.
- `toChainId`가 Hyperliquid로 설정된 경우, `toCurrency`가 지정되지 않으면 기본적으로 USDC로 설정됩니다.
- 0 주소는 자동으로 Hyperliquid USDC 형식으로 수정됩니다.

## 파트너 통합

파트너는 사용자 정의 설정으로 구성될 수 있습니다:

```typescript
interface PartnerConfig {
  returnToHomeUrl?: string;      // 기본 리디렉션 URL
  customRecipientLabel?: string; // 수신자에 대한 사용자 정의 레이블
  returnHomeLabel?: string;      // 홈으로 돌아가기 버튼의 사용자 정의 레이블
}
```

파트너로 등록하려면 AnySpend 팀에 연락하여 구성을 추가하십시오.

## 보안

입금 페이지는 내장된 보안 조치를 포함합니다:

- **리디렉션 URL 검증**: 오픈 리디렉션 공격을 방지하기 위해 HTTPS URL만 리디렉션에 허용됩니다.
- **금액 검증**: 금액은 유효한 양의 정수(wei 값)여야 합니다.
- **주소 검증**: 처리 전에 수신자 및 토큰 주소가 검증됩니다.

## 일반적인 사용 사례

- **텔레그램 봇**: 사용자가 간단한 링크를 통해 봇 지갑에 자금을 조달할 수 있습니다.
- **게임**: 플레이어가 게임 계정에 토큰을 입금할 수 있습니다.
- **마켓플레이스**: 구매자가 에스크로 주소에 자금을 조달할 수 있습니다.
- **구독 서비스**: 서비스 지갑에 반복적인 지불을 수집합니다.
- **팁 주기**: 공유 가능한 입금 링크로 창작자에게 팁을 줄 수 있습니다.

## 통합 예시

여기 귀하의 애플리케이션에서 입금 링크를 생성하는 방법이 있습니다:

```typescript
function generateDepositLink({
  recipientAddress,
  chainId,
  tokenAddress,
  amount,
  redirectUrl,
}: {
  recipientAddress: string;
  chainId: number;
  tokenAddress: string;
  amount?: string;
  redirectUrl?: string;
}) {
  const params = new URLSearchParams({
    recipientAddress,
    toChainId: chainId.toString(),
    toCurrency: tokenAddress,
  });

  if (amount) params.set("amount", amount);
  if (redirectUrl) params.set("redirect_url", redirectUrl);

  return `https://anyspend.b3.fun/deposit?${params.toString()}`;
}
```
