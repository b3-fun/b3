결제 링크는 고객이 지원되는 모든 체인에서 모든 토큰으로 결제할 수 있게 해주는 공유 가능한 URL입니다. 각 링크는 브랜딩, 양식, 배송 옵션 등으로 커스터마이징할 수 있습니다.

<Info>
  결제 링크는 AnySpend 프로토콜을 통해 토큰 라우팅, 가스 추상화 및 체인 간 스왑을 자동으로 처리합니다.
</Info>

## 결제 링크 객체

  결제 링크의 고유 식별자 (예: `pl_abc123`).
</ResponseField>

<ResponseField name="name" type="string">

<ResponseField name="form_schema" type="object | null">

  사용자 정의 브랜딩 구성.

<ResponseField name="items" type="array">

    <ResponseField name="id" type="string">항목 ID.</ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="created_at" type="string">

## URL 매개변수

결제 링크 URL을 공유할 때, 세션 데이터를 사전 채우거나, 속성을 추적하고, 고객과 결제를 연결하기 위해 쿼리 매개변수를 추가할 수 있습니다.

| 매개변수 | 설명 | 예시 |
|-----------|-------------|---------|
| `client_reference_id` | 내부 참조 ID (주문 ID, 사용자 ID 등) | `?client_reference_id=order_123` |
| `metadata[key]` | 임의의 키-값 메타데이터 (대괄호 표기법, 최대 50개 키) | `?metadata[user_id]=clerk_abc&metadata[plan]=pro` |
| `session_id` | 새로운 것을 생성하는 대신 사전 생성된 체크아웃 세션 사용 | `?session_id=cs_abc123` |
| `success_url` | 성공적인 결제 후 리디렉션 URL 재정의 | `?success_url=https://example.com/thanks` |
| `cancel_url` | 취소/뒤로 URL 재정의 | `?cancel_url=https://example.com/cart` |
| `utm_source` | 속성을 위한 UTM 소스 | `?utm_source=twitter` |
| `utm_medium` | 속성을 위한 UTM 매체 | `?utm_medium=social` |
| `utm_campaign` | 속성을 위한 UTM 캠페인 | `?utm_campaign=launch` |

### 메타데이터 제한

- 세션당 최대 50개 키
- 키 길이: 최대 40자
- 값 길이: 최대 500자

### 예시: 고객과 결제 연결

```
https://anyspend.com/pay/abc123?client_reference_id=order_456&metadata[user_id]=clerk_abc&metadata[plan]=pro
```

모든 매개변수는 백엔드에서 결제를 조정할 수 있도록 웹훅을 통해 전달됩니다.

<Tip>
  전체 제어가 필요한 엔터프라이즈 통합의 경우, `customer_email`, `customer_name`, `metadata`, 및 `client_reference_id`를 사용하여 서버 측에서 [Checkout Sessions API](/anyspend/api/checkout-sessions-api)를 통해 세션을 생성한 다음 반환된 `url`을 사용하여 리디렉션하거나 결제 링크에 `?session_id=`를 추가하세요.
</Tip>

## 결제 링크 목록

<ParamField query="search" type="string">
  이름이나 설명으로 결제 링크 검색.
</ParamField>

<ParamField query="active" type="boolean">
  활성 상태로 필터링. 모든 링크를 반환하려면 생략.
</ParamField>

<ParamField query="sort" type="string" default="created_at">
  정렬 필드. `created_at`, `updated_at`, `name`, `uses` 중 하나.
</ParamField>

<ParamField query="order" type="string" default="desc">
  정렬 순서. `asc`, `desc` 중 하나.
</ParamField>

<ParamField query="page" type="number" default={1}>
  페이지 번호를 위한 페이징.
</ParamField>

<ParamField query="limit" type="number" default={20}>
  페이지당 결과 수 (최대 100).
</ParamField>

<CodeGroup>
```bash cURL
curl -X GET "https://platform-api.anyspend.com/api/v1/payment-links?active=true&sort=created_at&order=desc&page=1&limit=20" \
  -H "Authorization: Bearer asp_xxx"
```

```typescript SDK
import { AnySpend } from "@b3dotfun/sdk";

const anyspend = new AnySpend(process.env.ANYSPEND_API_KEY!);

const { data, pagination } = await anyspend.paymentLinks.list({
  active: true,
  sort: "created_at",
  order: "desc",
  page: 1,
  limit: 20,
});
```
</CodeGroup>
