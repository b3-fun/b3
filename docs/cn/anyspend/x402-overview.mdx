## 什么是 AnySpend x402?

AnySpend x402 是 B3 实现的 **x402 支付协议**，它支持通过 HTTP 直接进行即时、自动的加密货币支付。与仅支持 USDC 的传统 x402 服务不同，**AnySpend 允许您使用任何代币支付**，同时资源服务器可以接收他们偏好的任何代币。

x402 协议使 HTTP `402 Payment Required` 状态码现代化，将其转变为机器对机器微支付、付费 API 和自治代理互动的实用解决方案。

## 对买家（支付方）

使用您持有的任何加密货币支付付费资源：

- **使用任何代币支付**：使用 ETH、DAI 或任何 ERC-20 代币 - 不仅仅是 USDC
- **标准协议**：使用标准 HTTP 与任何 x402 启用的服务兼容
- **无需手动兑换**：AnySpend 自动将您的代币转换为卖家需要的代币
- **简单集成**：一个 SDK 适用于所有 x402 服务

**使用案例：**
- 访问高级 API 和 AI 代理服务
- 支付计算资源和数据源
- 实现自治代理到代理的支付
- 以编程方式消费付费内容

## 对卖家（资源服务器）

接受 USDC 支付，而您的客户可以使用他们偏好的代币支付：

- **接收 USDC**：无论客户使用什么代币，都以 USDC 收款
- **Express 中间件**：只需几行代码即可将支付添加到您的 API
- **无需区块链基础设施**：AnySpend 处理所有链上复杂性
- **即时结算**：支付经验证并自动结算

**使用案例：**
- 通过高级 API 和数据服务实现盈利
- 构建付费 AI 代理工作流
- 创建付费内容和工具
- 无需加密专业知识即可接受加密支付

## 如何工作

<Steps>
<Step title="客户端请求资源">
用户的应用程序向您的 API 发出标准 HTTP 请求
</Step>

<Step title="需要支付">
您的 API 返回 `402 Payment Required` 并提供支付详情
</Step>

<Step title="代币偏好">
客户端指定他们偏好的支付代币（ETH、DAI 等）
</Step>

<Step title="AnySpend 报价">
AnySpend 计算出客户选择的代币的等值金额
</Step>

<Step title="客户端签署支付">
用户为他们的代币签署加密授权
</Step>

<Step title="自动结算">
AnySpend 将代币兑换为 USDC 并结算到您的钱包
</Step>

<Step title="资源交付">
您的 API 收到支付确认并返回请求的资源
</Step>
</Steps>

## 关键差异

### 多代币支持

**标准 x402**：资源服务器和客户端都必须使用 USDC

**AnySpend x402**：客户端用他们持有的**任何代币**支付。AnySpend 处理转换。

```
客户端支付 0.0003 ETH → AnySpend 兑换为 1 USDC → 卖家收到 1 USDC
```

### 无需基础设施

资源服务器无需：
- 运行区块链节点
- 管理钱包或私钥
- 处理代币兑换或桥接
- 监控交易状态

AnySpend 的服务商处理所有区块链操作，并提供用于验证和结算的简单 HTTP 端点。

### 标准协议合规

使用 AnySpend 的客户端**完全不知道**他们正在使用一个特殊的服务商。整个流程使用标准 x402 协议消息 - 多代币的魔法在资源服务器端通过中间件透明地发生。

## 由 AnySpend Facilitator 驱动

所有 x402 支付都通过 **AnySpend Facilitator** 处理 - 一个生产级支付基础设施，它处理：

- ✅ 跨 19+ 区块链网络的多代币支持
- ✅ 自动代币兑换和跨链路由
- ✅ 为用户提供无燃料费支付
- ✅ 符合标准的 x402 实现
- ✅ 任何 x402 服务商的即插即用替代品

**生产端点：** `https://mainnet.anyspend.com/x402`
