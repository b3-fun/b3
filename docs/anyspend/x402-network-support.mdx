### Other ERC-20 Tokens

Any ERC-20 token that implements **EIP-2612 permit** is supported:

- **cbBTC** (Coinbase Wrapped Bitcoin)
- **LINK** (Chainlink)
- **UNI** (Uniswap)
- **AAVE** (Aave)
- And many more...

**Check if a token supports permit:**
```typescript
// Check if token has permit function
const hasPermit = await token.read.permit !== undefined;

// Or check for DOMAIN_SEPARATOR (required for EIP-712)
const hasDomainSeparator = await token.read.DOMAIN_SEPARATOR !== undefined;
```

## Signature Methods

### EIP-3009: transferWithAuthorization

**Used by:** USDC (all networks)

**Advantages:**
- ✅ One-time use signature
- ✅ Gasless for payer
- ✅ Immediate execution (no approval needed)
- ✅ Random nonce (no sequential dependency)

**Message Structure:**
```solidity
transferWithAuthorization(
    address from,
    address to,
    uint256 value,
    uint256 validAfter,   // 0 (valid now)
    uint256 validBefore,  // deadline
    bytes32 nonce,        // random nonce
    bytes signature
)
```

**Example:**
```typescript
import { signTransferWithAuthorization } from 'anyspend-x402-client';

const signature = await signTransferWithAuthorization({
  from: payerAddress,
  to: recipientAddress,
  value: '1000000',              // 1 USDC
  validBefore: deadline,
  nonce: randomBytes32()
});
```

### EIP-2612: Permit

**Used by:** WETH, DAI, most modern ERC-20s

**Advantages:**
- ✅ Standard across many tokens
- ✅ Time-limited approvals
- ✅ Widely implemented
- ✅ Gasless for payer

**Message Structure:**
```solidity
permit(
    address owner,
    address spender,
    uint256 value,
    uint256 deadline,
    uint8 v, bytes32 r, bytes32 s
)
```

**Example:**
```typescript
import { signPermit } from 'anyspend-x402-client';

const nonce = await token.nonces(ownerAddress);

const signature = await signPermit({
  token: tokenAddress,
  owner: ownerAddress,
  spender: facilitatorAddress,
  value: '1000000000000000000',  // 1 DAI
  deadline: deadline,
  nonce: nonce
});
```

## Token Configuration

### Client-Side (Paying)

Specify your preferred payment token:

```typescript
import { X402Client } from 'anyspend-x402-client';

const client = new X402Client({
  walletClient,
  preferredToken: '0x4200000000000000000000000000000000000006', // WETH on Base
  network: 'base-mainnet'
});
```

Or per-request:

```typescript
const response = await client.request('https://api.example.com/data', {
  preferredToken: '0x50c5725949A6F0c72E6C4a641F24049A917DB0Cb' // DAI on Base
});
```

### Server-Side (Receiving)

Configure accepted tokens and settlement currency:

```typescript
import { paymentMiddleware, facilitator } from '@b3dotfun/anyspend-x402';

const endpointConfig = {
  '/api/data': {
    amount: '1.00',        // $1 worth of any token
    asset: 'USD',          // Let client choose token
    network: 'base-mainnet'
  }
};

app.use(paymentMiddleware(
  '0xYourUSDCAddress...',  // Receive USDC
  endpointConfig,
  facilitator
));
```

## Cross-Chain Support

AnySpend can facilitate payments where the **client's token** is on a different chain than the **resource server's settlement**.

**Example:**
```typescript
// Client pays with WETH on Ethereum
const clientPayment = {
  token: '0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2', // WETH on Ethereum
  network: 'ethereum-mainnet',
  amount: '0.0003'
};

// Resource server receives USDC on Base
const serverSettlement = {
  token: '0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913', // USDC on Base
  network: 'base-mainnet',
  amount: '1.00'
};

// AnySpend handles:
// 1. Receive WETH on Ethereum
// 2. Bridge WETH from Ethereum → Base
// 3. Swap WETH → USDC on Base
// 4. Settle USDC to resource server
```

**Supported Cross-Chain Routes:**
- Ethereum ↔ Base
- Ethereum ↔ Arbitrum
- Ethereum ↔ Optimism
- Ethereum ↔ Polygon
- Base ↔ Arbitrum
- And more...

<Info>
Cross-chain payments take longer (2-5 minutes) due to bridging time and incur additional fees (~0.5% total).
</Info>

## Network Selection Guide

### Base (Recommended)

**Best for:**
- High-frequency micropayments
- Low-value transactions ($0.01 - $10)
- Real-time APIs and services

**Pros:**
- Extremely low gas fees (~$0.001 per transaction)
- Fast block times (~2 seconds)
- Growing ecosystem
- Coinbase-backed security

**Cons:**
- Smaller liquidity than Ethereum mainnet
- Newer chain with less battle-testing

### Ethereum Mainnet

**Best for:**
- High-value transactions ($100+)
- Maximum security requirements
- Largest token selection

**Pros:**
- Maximum liquidity
- Most secure and battle-tested
- Widest token support
- Industry standard

**Cons:**
- High gas fees ($5-50 per transaction)
- Slower block times (~12 seconds)
- Not economical for micropayments

### Arbitrum One

**Best for:**
- Medium-value transactions ($1-100)
- DeFi integrations
- Balance of cost and security

**Pros:**
- Very low gas fees
- Fast finality
- Good liquidity
- Ethereum-equivalent security

**Cons:**
- Withdrawal delays (7 days to mainnet)
- Less mainstream than Base

### Optimism

**Best for:**
- Similar use cases as Arbitrum
- Projects already on OP Stack

**Pros:**
- Low fees
- Fast transactions
- Ethereum-aligned roadmap
- Growing ecosystem

**Cons:**
- Similar limitations to Arbitrum

### Polygon PoS

**Best for:**
- Ultra-low-cost transactions
- Gaming and NFT use cases
- Emerging markets

**Pros:**
- Extremely low fees (~$0.0001 per tx)
- Fast transactions
- Large user base
- Mature ecosystem

**Cons:**
- Different security model (not L2)
- Bridge concerns
- MATIC token required for gas

## Gas Fees Comparison

Estimated gas costs per X402 payment (as of 2024):

| Network | USDC Payment | Token Swap Payment | Cross-Chain Payment |
|---------|-------------|-------------------|---------------------|
| Base | ~$0.001 | ~$0.003 | ~$0.01 - $0.05 |
| Ethereum | ~$5 - $50 | ~$10 - $100 | ~$20 - $200 |
| Arbitrum | ~$0.01 | ~$0.05 | ~$0.1 - $1 |
| Optimism | ~$0.01 | ~$0.05 | ~$0.1 - $1 |
| Polygon | ~$0.0001 | ~$0.001 | ~$0.01 - $0.1 |

<Note>
Gas fees are **paid by the facilitator** and included in the 0.25% - 0.35% AnySpend fee. Clients and resource servers don't pay gas directly.
</Note>
