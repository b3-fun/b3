## Regras de Validação por IA

O endpoint de validação por IA avalia sua ideia de artigo e a aprova como está, sugere melhorias ou a rejeita.

<Tabs>
  <Tab title="✅ Pontuação ≥80">
    **Perfeito!** Sua entrada é aprovada como está.
    
    Seu título ou tópico fornecido é excelente e se encaixa no foco do site. Não são necessárias alterações - enviaremos exatamente como você escreveu.
    
    **Exemplo:** "Como Usar o Base Pay para Criadores no Seu DApp"
  </Tab>
  <Tab title="⚠️ Pontuação 70-79">
    **Bom com Sugestões!** IA fornece uma versão melhorada.
    
    Seu conteúdo é relevante, mas poderia ser otimizado. IA gera um título e descrição melhores, mantendo sua ideia central.
    
    **Entrada de Exemplo:** "base pay para criadores"
    
    **Saída da IA:** "Integrando o Base Pay: Um Guia Completo para Criadores de Conteúdo"
  </Tab>
  <Tab title="❌ Pontuação <70">
    **Rejeitado.** Conteúdo não se encaixa no foco do site.
    
    Seu tópico está fora do tema para este site. Tente um tópico diferente ou escolha um site diferente que corresponda melhor ao seu conteúdo.
    
    **Exemplo:** Enviando uma receita de cozinha para um site de notícias cripto
  </Tab>
</Tabs>

### Validar Ideia de Artigo (Grátis)

```bash
POST https://x402.anyspend.com/articles/validate
```

Endpoint gratuito para validar sua ideia de artigo. IA gera um título, descrição e pontuação de aprovação.

**Corpo da Requisição:**

<CodeGroup>
```json Request
{
  "siteDomain": "baseradar.com",
  "userInput": "How to use Base Pay for creators"
}
```

```json Resposta de Sucesso (200) - Aprovado
{
  "success": true,
  "data": {
    "approved": true,
    "score": 85,
    "reasoning": "Bom tópico para site focado em Base. IA otimizou o título para melhor engajamento.",
    "suggestedTitle": "Integrando o Base Pay: Um Guia Completo para Criadores de Conteúdo",
    "suggestedDescription": "Aprenda como implementar o Base Pay na sua plataforma de criadores com exemplos passo a passo e melhores práticas.",
    "estimatedPrice": "50000000",
    "userProvidedTitle": false,
    "message": "Conteúdo aprovado com sugestões da IA"
  }
}
```

```json Resposta de Sucesso (200) - Pontuação Perfeita
{
  "success": true,
  "data": {
    "approved": true,
    "score": 95,
    "reasoning": "Título excelente! Claro, específico e altamente relevante para o público do site.",
```

**Parâmetros:**
- `siteDomain` (obrigatório) - Domínio do site alvo (ex.: "baseradar.com")
- `userInput` (obrigatório) - URL de tweet, ideia de tópico ou título completo (10-200 caracteres)

### Enviar Solicitação de Artigo (Pagamento x402)

```bash
POST https://x402.anyspend.com/articles/buy
```

Envie uma solicitação de artigo com pagamento x402. Você deve validar seu conteúdo primeiro e receber aprovação.

**Corpo da Requisição:**

<CodeGroup>
```json Request
{
  "buyerAddress": "0x742d35Cc6...",
  "siteDomain": "baseradar.com",
  "title": "Integrando o Base Pay: Um Guia Completo para Criadores de Conteúdo",
  "description": "Aprenda como implementar o Base Pay na sua plataforma de criadores com exemplos passo a passo e melhores práticas."
}
```

```json Resposta de Sucesso (200)
{
  "success": true,
  "data": {
    "requestId": "req_789...",
    "productSpecificId": "req_789...",
    "totalPrice": "50000000",
    "status": "pending"
  }
}
```

```json Resposta de Erro (400)
{
  "success": false,
  "error": "Validação de conteúdo necessária antes da submissão"
}
```
</CodeGroup>

**Parâmetros:**
- `buyerAddress` (obrigatório) - Seu endereço de carteira
- `siteDomain` (obrigatório) - Domínio do site alvo
- `title` (obrigatório) - Título do artigo (da validação ou o seu próprio se pontuação ≥90)
- `description` (obrigatório) - Descrição do artigo (da validação ou a sua própria)

**Exemplo em JavaScript/TypeScript:**

```typescript
import { wrapFetchWithPayment } from '@b3dotfun/anyspend-x402-fetch';
import { createWalletClient, custom, publicActions } from 'viem';
import { base } from 'viem/chains';

// Passo 1: Valide seu conteúdo (GRÁTIS)
const validateResponse = await fetch('https://x402.anyspend.com/articles/validate', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    siteDomain: 'baseradar.com',
    userInput: 'How to use Base Pay for creators'
  })
});

const validation = await validateResponse.json();

if (!validation.data.approved) {
  console.error('Conteúdo rejeitado:', validation.data.reasoning);
  process.exit(1);
}

console.log('✅ Aprovado! Pontuação:', validation.data.score);
console.log('Título:', validation.data.suggestedTitle);

// Passo 2: Enviar com pagamento (x402)
const walletClient = createWalletClient({
  account: '0x742d35Cc6...',
  chain: base,
  transport: custom(window.ethereum),
}).extend(publicActions);

const fetchWithPayment = wrapFetchWithPayment(
  fetch,
  walletClient,
  BigInt(1000 * 10 ** 6) // Máximo de 1000 USDC
);

const response = await fetchWithPayment(
  'https://x402.anyspend.com/articles/buy',
  {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      buyerAddress: '0x742d35Cc6...',
      siteDomain: 'baseradar.com',
      title: validation.data.suggestedTitle,
      description: validation.data.suggestedDescription
    })
  }
);

const result = await response.json();
console.log('🎉 Artigo enviado! ID da Solicitação:', result.data.requestId);
```

<Note>
  Veja a [documentação do SDK x402](/anyspend/x402-overview) para mais detalhes sobre o uso de `wrapFetchWithPayment`.
</Note>

## Dicas de Validação por IA

<AccordionGroup>
  <Accordion title="Escreva Títulos Claros e Específicos">
    **Bom:** "Como Construir um DEX no Base com Uniswap V3"
    
    **Ruim:** "Coisas de DeFi"
    
    Títulos específicos recebem pontuações mais altas e têm mais chances de serem aprovados.
  </Accordion>
  
  <Accordion title="Combine com o Foco do Site">
    Verifique as palavras-chave e descrição do site antes de enviar. Um artigo focado em Base não terá boa pontuação em um site exclusivo de Solana.
  </Accordion>
  
  <Accordion title="Use Frases Completas para Títulos">
    **Bom:** "Como Usar o Base Pay para Criadores no Seu DApp"
    
    **Ruim:** "base pay criadores"
    
    Títulos completos (10-200 caracteres) recebem pontuações mais altas.
  </Accordion>
  
  <Accordion title="Aproveite URLs de Tweets">
    Se você tem um tweet viral sobre seu tópico, use a URL! IA extrai o contexto e gera um título relevante.
  </Accordion>
</AccordionGroup>

## Problemas Comuns

<AccordionGroup>
  <Accordion title="'Conteúdo rejeitado' com pontuação baixa">
    Seu tópico não se encaixa no foco do site. Opções:
    - Escolha um site diferente que combine melhor com seu tópico
    - Ajuste seu tópico para alinhar com as palavras-chave do site
    - Forneça um título mais específico e detalhado
  </Accordion>
  
  <Accordion title="Erro 'Validação de conteúdo necessária'">
    Você deve chamar `/articles/validate` primeiro e obter aprovação antes de chamar `/articles/buy`.
  </Accordion>
  
  <Accordion title="Erro 'Saldo insuficiente'">
    Você precisa de USDC na rede Base. Verifique
