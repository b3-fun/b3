Links de pagamento são URLs compartilháveis que permitem aos seus clientes pagar com qualquer token em qualquer cadeia suportada. Cada link pode ser personalizado com marca, formulários, opções de envio e mais.

<Info>
  Links de pagamento lidam automaticamente com roteamento de token, abstração de gás e trocas entre cadeias via o protocolo AnySpend.
</Info>

## Parâmetros da URL

Ao compartilhar uma URL de link de pagamento, você pode anexar parâmetros de consulta para pré-preencher dados da sessão, rastrear atribuição e associar pagamentos com seus clientes.

| Parâmetro | Descrição | Exemplo |
|-----------|-------------|---------|
| `client_reference_id` | Seu ID de referência interno (ID do pedido, ID do usuário, etc.) | `?client_reference_id=order_123` |
| `metadata[key]` | Metadados chave-valor arbitrários (notação de colchetes, até 50 chaves) | `?metadata[user_id]=clerk_abc&metadata[plan]=pro` |
| `session_id` | Use uma sessão de checkout pré-criada em vez de criar uma nova | `?session_id=cs_abc123` |
| `success_url` | Substitua a URL de redirecionamento após o pagamento bem-sucedido | `?success_url=https://example.com/thanks` |
| `cancel_url` | Substitua a URL de cancelamento/volta | `?cancel_url=https://example.com/cart` |
| `utm_source` | Fonte UTM para atribuição | `?utm_source=twitter` |
| `utm_medium` | Meio UTM para atribuição | `?utm_medium=social` |
| `utm_campaign` | Campanha UTM para atribuição | `?utm_campaign=launch` |

### Limites de metadados

- Máximo de 50 chaves por sessão
- Comprimento da chave: até 40 caracteres
- Comprimento do valor: até 500 caracteres

### Exemplo: associando pagamentos com seus clientes

```
https://anyspend.com/pay/abc123?client_reference_id=order_456&metadata[user_id]=clerk_abc&metadata[plan]=pro
```

Todos os parâmetros são passados para webhooks, então você pode conciliar pagamentos no seu backend.

<Tip>
  Para integrações empresariais que precisam de controle total, use a [API de Sessões de Checkout](/anyspend/api/checkout-sessions-api) para criar sessões no lado do servidor com `customer_email`, `customer_name`, `metadata`, e `client_reference_id`, depois redirecione usando a `url` retornada ou anexe `?session_id=` ao seu link de pagamento.
</Tip>

## Listar links de pagamento

<ParamField query="search" type="string">
  Pesquise links de pagamento por nome ou descrição.
</ParamField>

<ParamField query="active" type="boolean">
  Filtrar pelo status ativo. Omita para retornar todos os links.
</ParamField>

<ParamField query="sort" type="string" default="created_at">
  Campo de ordenação. Um de: `created_at`, `updated_at`, `name`, `uses`.
</ParamField>

<ParamField query="order" type="string" default="desc">
  Ordem de classificação. Um de: `asc`, `desc`.
</ParamField>

<ParamField query="page" type="number" default={1}>
  Número da página para paginação.
</ParamField>

<ParamField query="limit" type="number" default={20}>
  Número de resultados por página (máx. 100).
</ParamField>

<CodeGroup>
```bash cURL
curl -X GET "https://platform-api.anyspend.com/api/v1/payment-links?active=true&sort=created_at&order=desc&page=1&limit=20" \
  -H "Authorization: Bearer asp_xxx"
```

```typescript SDK
import { AnySpend } from "@b3dotfun/sdk";

const anyspend = new AnySpend(process.env.ANYSPEND_API_KEY!);

const { data, pagination } = await anyspend.paymentLinks.list({
  active: true,
  sort: "created_at",
  order: "desc",
  page: 1,
  limit: 20,
});
```
