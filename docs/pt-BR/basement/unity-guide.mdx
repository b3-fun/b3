## Exemplos de Código

### Acionar o Motor de Regras

Aqui está um exemplo de como acionar o motor de regras dentro do seu jogo Unity:

```csharp Unity C# Example
B3LauncherClient.TriggerRulesEngine(new B3LauncherClient.TriggerRulesEngineBody
{
    launcherJwt = B3Instance.Instance.SessionJWT ?? jwtInput.text,
    trigger = "open-tip-modal",
    otherWallet = otherWalletInput.text,
}, null);
```

### Gerenciamento de Sessão

O SDK gerencia automaticamente as sessões do usuário:

```csharp Session Example
// Acessar o JWT da sessão atual
string sessionToken = B3Instance.Instance.SessionJWT;

// Verificar se o usuário está autenticado
if (B3Instance.Instance.IsAuthenticated)
{
    // Usuário está logado e pronto para interagir com os serviços B3
    Debug.Log("Usuário autenticado com sucesso");
}
```

## Integração de API

O Unity SDK oferece acesso transparente a todas as APIs do launcher B3:

<AccordionGroup>
<Accordion title="Chamadas Baseadas em Callback">
  Padrão de callback tradicional para lidar com respostas de API:
  
  ```csharp
  B3LauncherClient.SomeAPICall(requestData, (response) => {
      // Lidar com a resposta
      Debug.Log("Chamada de API concluída");
  });
  ```
</Accordion>

<Accordion title="Chamadas Async/Await">
  Padrão moderno async/await para um código mais limpo:
  
  ```csharp
  public async void CallAPI()
  {
      var response = await B3LauncherClient.SomeAPICallAsync(requestData);
      // Lidar com a resposta
      Debug.Log("Chamada de API concluída");
  }
  ```
</Accordion>

<Accordion title="Webhooks WebGL">
  Gerenciar eventos do launcher em builds WebGL:
  
  ```csharp
  // Registrar manipuladores de webhook
  B3Instance.Instance.OnWebhookReceived += HandleWebhook;
  
  private void HandleWebhook(WebhookData data)
  {
      // Processar os dados do webhook recebido
      Debug.Log($"Webhook recebido: {data.type}");
  }
  ```
</Accordion>
</AccordionGroup>
